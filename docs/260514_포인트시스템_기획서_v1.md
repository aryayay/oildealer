## 개요

### 한 줄 요약

> 오일딜러 앱 활동(로그인·댓글·공유·초대)과 Board51 광고 매체 부착에 포인트를 지급, 주유소 사장·소장이 MRO 소모품을 포인트로 구매할 수 있는 시스템 도입. 포인트는 개인에게 귀속.

### 핵심 가치

- 주유소 사장·소장: 광고 매체 부착 보상 + 앱 활동 보상을 개인 포인트로 받음
- 비즈니스: 앱 활성화·에이전트 연결 전환·MRO 시장 진입을 하나의 인센티브 시스템으로 묶음
- 운영: 비닐장갑 박스 현물 지급 폐지로 운영 단순화

---

## 도입 배경 및 목적

### 배경

- 현재 Board51 광고 게첨 주유소에는 비닐장갑 박스로 현물 리워드 지급 중
- 오일딜러 앱은 자체 활성화 인센티브가 없는 상태
- MRO 소모품 사업이 운영팀 최종 로드맵

### 목적

| 목적 | 강화할 행동 | 측정 KPI |
|---|---|---|
| 앱 활성화 | 일일 진입·참여 | DAU/WAU |
| 에이전트 연결 전환 | B 상태 → C 상태 | 연결률 |
| 광고 리워드 디지털화 | 매체 부착·집행 | Board51 인벤토리 사용률 |
| MRO 시장 진입 | 포인트로 소모품 구매 | 객단가, 재구매율 |

## 정책 정의

### 적립 정책

#### A. 매체 적립 (Board51 광고 게첨 주유소)

- 비닐장갑 박스 현물 지급 → 폐지
- 어드민에서 운영팀이 수동으로 포인트 지급
- 지급 기준·금액은 운영팀 내부 정책에 따름 (기획서 외)
- 귀속: 사장 개인에게 적립

#### B. 온보딩 (1회성 보너스)

| 행동 | 포인트 | 귀속 |
|---|---|---|
| 신규 가입 완료 | 1,000P | 본인 |
| 에이전트 연결 요청 | 10,000P | 작업자 본인 |
| 초대받아 가입 (피초대인) | 2,000P | 피초대인 본인 |
| 추천인 보너스 (피초대인 가입) | 2,000P | 추천인 본인 |
| 추천인 보너스 (피초대인 연결) | 3,000P | 추천인 본인 |

#### C. 앱 활동 (반복 적립)

| 행동 | 포인트 | 한도 | 위치 |
|---|---|---|---|
| 로그인 (30초 이상 체류) | 200P | 1일 1회 | 전역 |
| 전망 글 좋아요 | 50P | 1일 5회 | 시장 → 전망 |
| 사장님들 한마디 댓글 | 300P | 1일 1회 | 시장 → 전망 |
| 외부 공유 (링크 열림 1회 이상) | 500P | 주 2회 | 전역 |
| 초대 메시지 발송 | 200P | 주 5회 | 전역 |

전 직무(사장·소장) 동일 적용.

#### 주간 적립 시뮬레이션

| 행동 | 주 합계 |
|---|---|
| 로그인 7일 | 1,400P |
| 좋아요 일 5회 × 7일 | 1,750P |
| 한마디 댓글 7일 | 2,100P |
| 공유 2회 | 1,000P |
| 초대 5회 | 1,000P |
| **합계** | **7,250P** |

평범 사용자(로그인 + 댓글만): 주 약 3,500P

#### 적립 제외

- 거래(유류 주문) 자체 적립 없음
- 거래 후 리뷰 시스템 없음

### 사용 정책

- 사용처: MRO 카탈로그 단독
- 환산: 1P = 1원 (어드민 변수)
- 부분 결제 불가
- 잔액 부족 시: "포인트 OOOP 부족합니다" 안내 + "포인트는 이렇게 얻을 수 있어요" 적립 가이드 노출 (외부몰 링크 없음)
- 유효기간 없음 (만료 정책 미적용)
- 양도/이관: 개인 귀속, 이직 시 본인 계정 유지
- 현금화 불가
- 환불 시 본인 포인트로 복원

### 일일·월간 한도

한도 없음. 어뷰징 데이터 모니터링 후 V2에서 재검토.

### 포인트 귀속 정책

| 적립 유형 | 귀속 |
|---|---|
| 매체 적립 | 사장 개인 |
| 온보딩 | 작업자 본인 |
| 앱 활동 | 본인 |

- 개인 귀속 원칙, 이직·퇴사 시 본인 계정·포인트 유지
- 사장 변경 시: 별도 정책 필요 (잔여 결정사항)

### MRO 사용 흐름

```
[사장·소장: 앱에서 MRO 상품 주문]
   ↓ 포인트 결제 → 즉시 차감
[어드민 주문 큐에 적재]
   ↓ 하루 1회 운영팀 일괄 처리
[운영팀이 매입처에서 일괄 발주]
   ↓ 송장 회수
[어드민에 송장 입력 → 사용자 푸시]
   ↓
[사용자 주유소로 배송]
```

## 어드민 컨트롤 범위

원칙: MRO 운영 관련 모든 변수는 어드민에서 운영팀이 직접 조정.

### A. 상품 (카탈로그)

- 상품 추가/수정/삭제 (단건·CSV)
- 상품명/규격/이미지/설명
- 가격(P) 자유 설정
- 매입가(원) 내부 메모
- 카테고리
- 재고 수량
- 노출 on/off
- 추천 상품 플래그 (홈 위젯 노출 대상)
- 정렬 우선순위 (인기순 점수)

### B. 가격 (포인트)

- 1P = 환산 가격 (기본 1원)
- 상품별 가격(P) 일괄 조정 (%/정액)
- 시즌·이벤트 가격 (시작·종료일 자동 적용)

### C. 배송 정책

- 기본 배송비 (조정 가능)
- 무료 배송 기준 (조정 가능)
- 상품별 배송비 override
- 지역별 배송비 (도서산간 추가)
- 일괄 처리 시각 (기본 매일 오후 3시)

### D. 주문 / 발송 관리

- 주문 상태 변경
- 송장 입력 (단건·CSV)
- 환불 (포인트 복원)
- SLA 라벨 (24시간 미승인 빨강)

### E. 포인트 시스템 변수

- 매체 적립 수동 지급
- 사용자별 수동 적립·회수
- 적립 룰 비활성화 토글

### F. 회원 / 권한

- 사용자 검색·필터
- 강제 탈퇴
- 어뷰징 의심 큐

## 화면 기획

### 전체 화면 맵

#### 고객용 앱

```
홈
├─ 포인트 + 미니 상품 위젯 (사장·소장)
└─ 시장 정보

시장
├─ 시황
└─ 전망
   ├─ 전망 글 (좋아요·공유 적립)
   └─ 사장님들 한마디 (댓글 적립)

MRO 메인 화면
├─ 상단: 잔액·내역 진입
└─ 하단: MRO 상품 그리드 (카테고리 필터, 인기순 기본)

마이페이지
├─ 포인트 내역
├─ 친구 초대
└─ MRO 주문 내역

상세
├─ MRO 상품 상세 (잔액 부족 시 적립 가이드)
├─ MRO 결제 → 주문 완료
└─ 적립 내역 / 사용 내역

가입 흐름
├─ 직무 선택 (사장 / 소장)
└─ 직무별 인증 분기
```

#### 어드민

```
포인트 관리
├─ 사용자별 포인트 현황
├─ 수동 적립/회수
├─ 매체 적립 수동 입력
└─ 어뷰징 의심 큐

MRO 관리
├─ 카탈로그 등록/수정 (A-04)
├─ 카테고리 관리 (A-10)
├─ 주문 처리 큐 (A-05)
└─ 송장 입력

배송·정책 관리 (A-09)
├─ 배송비 / 무료 배송 기준
├─ 지역별 배송비
└─ 일괄 처리 시각

회원 관리
└─ 사장-소장 관계도, 강제 탈퇴

시스템 변수 (A-11)
└─ 적립 룰 토글·수치
```

#### 홈 위젯 (사장·소장 노출)

**구성**:
1. 상단 라인: 보유 포인트 + "MRO" 라벨 + 화살표
2. 미니 상품 그리드 (가로 스크롤): 어드민이 추천 플래그 설정한 상품 3~4개

**노출 알고리즘**: 어드민 추천 플래그 ON 상품 중 어드민 정렬 점수 순. 운영팀이 수동 큐레이션.

#### C-01. MRO 메인 화면

**구성**:
1. 상단 카드: 보유 포인트 (큰 숫자)
2. 빠른 메뉴: [적립 내역] [사용 내역] [친구 초대]
3. 카테고리 필터: 어드민 등록 카테고리 동적
4. MRO 상품 그리드 (2열)
5. 정렬 기본값: 인기순 (어드민 정렬 점수)
6. 정렬 옵션: 인기순 / 낮은 가격순 / 높은 가격순

#### C-05. MRO 상품 상세

**구성**:
1. 상품 이미지 (스와이프)
2. 상품명·규격 / 가격(P) / 재고
3. 잔액 비교 박스:
   - 잔액 ≥ 가격: "구매 가능 (잔액 [X]P)"
   - 잔액 < 가격: "포인트 [Y]P 부족합니다"
4. 잔액 부족 시 하단: "포인트는 이렇게 얻을 수 있어요" 적립 가이드 카드
   - 매일 로그인 200P
   - 전망 좋아요/댓글
   - 친구 초대
5. 상품 설명 / 배송 / 반품
6. 하단 CTA: [포인트로 구매] (잔액 충분 시만 활성화)

#### C-06. MRO 결제 화면

**구성**:
1. 주문 상품 요약
2. 배송지 (사업장 주소 자동, 수정 가능)
3. 결제 정보: 상품가 + 배송비 (어드민 변수) - 결제 후 잔액
4. 약관 동의
5. 하단 CTA: [Y P로 결제하기]

#### C-11. 가입 - 직무 선택

**구성**:
1. "어떤 자격으로 가입하시나요?"
2. 2개 선택 카드:
   - 사장: 사업자등록증상 대표
   - 소장: 운영 책임자
3. 직무별 다음 단계:
   - 사장: 사업자번호 + 대표자명 인증
   - 소장: 사장 승인 대기 또는 초대 링크

#### A-03. 매체 적립 수동 입력

**구성**:
1. 주유소 검색
2. 사장 계정 자동 매칭
3. 적립 금액 입력
4. 사유 메모 필수
5. 일괄 처리 (CSV)
6. 처리 후 사장 푸시

#### A-04. MRO 카탈로그 관리

**구성**:
1. 상품 리스트 (이름·카테고리·가격·매입가·재고·노출·추천 플래그·수정일)
2. 상단 액션: [+상품 추가] [CSV 일괄] [선택 노출 on/off] [선택 가격 일괄 변경] [선택 카테고리 일괄 변경] [선택 추천 on/off]
3. 상품 등록 모달: 상품명·카테고리·규격·가격(P)·매입가·이미지·설명·배송비 override·재고·노출·추천·정렬 우선순위
4. 시즌·이벤트 가격 설정 (시작·종료일)

#### A-05. MRO 주문 처리 큐

**구성**:
1. 상태 탭: [요청] [발주대기] [발주완료] [배송중] [완료] [취소]
2. 주문 리스트 (주문번호·주유소·구매자·상품·수량·금액·배송지·신청일·SLA)
3. 일괄 액션: 발주완료 표시 / 송장 CSV 업로드 / 배송중 / 완료
4. 개별 주문: 송장 입력·상태 변경·환불 (포인트 복원)
5. SLA 24시간 미승인 빨강 라벨

#### A-09. 배송·정책 관리

**구성**:
1. 기본 배송비 + 무료 배송 기준
2. 지역별 추가 배송비
3. 상품별 override
4. 일괄 처리 시각
5. SLA 정책 (발주·배송·품절 환불)
6. 변경 로그

#### A-10. 카테고리 관리

**구성**:
1. 카테고리 리스트 (이름·SKU 수·순서·노출)
2. 추가/수정/삭제
3. 드래그 순서 변경

#### A-11. 시스템 변수

**구성**:
1. 1P 환산 가격
2. 적립 룰 수치·토글 (로그인·좋아요·댓글·공유·초대)
3. 변경 로그

## 시나리오 흐름

### 시나리오 A. 사장 신규 가입 → 매체 적립 → 첫 MRO 구매

```
[사장 가입 → 사업자번호 인증]
   ↓ +1,000P
[홈 진입 → 포인트+상품 미니 위젯 노출]
   ↓
[1주일 활동: 로그인·댓글·공유 → 약 5,000P 누적]
   ↓
[월말 어드민 매체 적립 수동 입력 → 사장 계정 적립]
   ↓
[MRO 메인 화면 → 비닐장갑 49,500P 구매]
   ↓ 즉시 차감 + 배송비 자동 적용
[어드민 주문 큐 → 운영팀 오후 3시 일괄 발주]
   ↓
[송장 입력 → 배송 완료]
```

### 시나리오 B. 잔액 부족 → 적립 가이드 → 재진입

```
[소장 잔액 2,000P, 워셔액 3,800P 상품 진입]
   ↓
[C-05 상품 상세 → "포인트 1,800P 부족" 노출]
   ↓
[하단 적립 가이드 카드: "매일 로그인 200P / 댓글 300P / 공유 500P"]
   ↓
[소장이 며칠 활동 → 4,000P 달성]
   ↓
[재진입 → 구매 완료]
```

### 시나리오 C. 어드민 가격 인상

```
[운영자: A-04에서 워셔액 3,800P → 4,200P 인상]
   ↓ 즉시 반영
[고객용 앱 MRO 메인 화면 → 가격 자동 갱신]
```

### 시나리오 D. 사장/소장 이직

```
[소장 A — 주유소 X / 잔액 50,000P]
   ↓
[A가 주유소 X 퇴사 → 본인 계정 + 포인트 유지]
   ↓
[A가 주유소 Y에 소장으로 신규 가입]
   ↓
[A는 Y에서도 본인 포인트 + 신규 적립 사용 가능]
```

## 어뷰징 가드레일

| 행동 | 가드레일 |
|---|---|
| 로그인 | 30초 이상 체류 / 1 계정 = 1 디바이스 |
| 댓글 | 최소 10자 / 동일 텍스트 반복 차단 / 신고 누적 시 회수 |
| 좋아요 | 동일 글 1회 한정 / 취소 시 회수 |
| 공유 | unique 토큰, 외부 1회 이상 열림 시 적립 |
| 초대 | 동일 번호/이메일 중복 차단, 30일 후 한도 회복 |
| 매체 적립 | 사장 단독 수령 |
| MRO 주문 | 사장·소장만 |
| 다계정 | 1 사업자 × 1 직무 = 1 계정 |

일일·월간 총량 한도는 미적용. V2에서 어뷰징 데이터 모니터링 후 도입 검토.

## 운영 변경사항

### 큐브플래닛 / Board51 측

- 비닐장갑 박스 현물 지급 트랙 폐지 (사후 통보)
- 광고 게첨 주유소 정보는 운영팀이 어드민에 직접 입력

### 오일딜러 운영

- MRO 재고 미보유 — 운영팀 일괄 발주
- MRO 카탈로그·가격·배송 정책 전체 어드민 컨트롤
- 매일 오후 3시 일괄 처리
- 매체 적립: 어드민 수동
- 세금계산서: 오일딜러 법인 발행

### 회원 / 권한 관리

- MVP 가입 직무: 사장·소장 2개
- 사장 인증: 사업자번호 + 대표자명 일치
- 소장 임명: 사장 권한
- 1 사업자 × 1 직무 = 1 계정
