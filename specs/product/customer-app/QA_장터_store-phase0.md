# 장터(MRO) QA 트래커 — feature/store-phase0

> 🧪 장터 프리뷰(`feature/store-phase0`) 테스트 중 발견한 QA·수정요청을 모으는 로그.
> 상태: `열림`(요청) → `수정중`(개발) → `검증대기`(push됨, Arya 확인 전) → `완료`(확인됨)
> 관련 spec: `07_장터_기획서_v1`(고객) · `admin/09_장터어드민`(어드민)

| ID | 일자 | 화면 | 요청 내용 | 유형 | 상태 | 커밋/비고 |
|---|---|---|---|---|---|---|
| QA-01 | 2026-06-25 | 신청(StoreApply) | 배송지 기본=주유소 주소. '다른 배송지' 변경 버튼 + **도로명 주소 검색(주소/우편번호 API)**으로 배송지 수정 가능해야 함 (현재 단순 텍스트) | 개발검토 | 열림 | Snoo 요청 |
| QA-02 | 2026-06-25 | 어드민 입금관리 | 장터 신청 입금을 **어드민 입금관리에서 확인** 가능하게. 현재 장터 설정에 입금계좌 미등록이라 동작 안 함 → ① 입금계좌 등록·활성화 ② 입금건 확인 처리 | 개발검토 | 열림 | Snoo 요청 |

---

## 사용법
- 새 QA 발견 → 행 추가(`열림`).
- Snoo가 수정 push → `검증대기` + 커밋 기재.
- Arya 프리뷰 재확인 OK → `완료`.
